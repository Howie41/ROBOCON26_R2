# ROBOCON26 R2 梅林工程总览

## 1. 工程目标

本工程当前围绕 `ROBOCON26_R2` 的两类能力展开：

1. 非梅林阶段
- 底盘保持低位模式
- 接收上位机 / 雷达给出的目标坐标
- 在平地完成导航移动
- 无移动指令时尽量保持底盘稳定，不乱跑

2. 梅林阶段
- 在 4 x 3 的梅林台阶区域内完成逐级上台阶、下台阶
- 支持旋转 90 度后的继续上下台阶
- 使用“雷达粗定位 + 激光/光电细判定”完成动作切换

当前已经在实车上验证通过的目标是：

- 可以完整走通整个 4 x 3 梅林流程
- `LB / RB` 旋转 90 度后，`heading` 会跟着更新
- `View` 可以识别当前位于哪一个梅林格子
- `Y` 可以沿当前车头方向上一个台阶
- `A` 可以沿当前车尾方向下一个台阶

## 2. 机械结构

### 2.1 底盘

- 四个 45 度安装的全向轮
- 底盘支持 `x / y / yaw` 平面运动

### 2.2 前腿与上台阶机构

- 前面有两条较短前腿
- 两个 2006 电机负责高位推进 / 后退
- 进入高位后，2006 负责带动整车跨台阶

### 2.3 后腿 / 后支撑

- 后部为较长的从动支撑
- 后部没有主动驱动能力

### 2.4 抬升

- 两个 3508 负责高低位切换
- 车身当前有两个关键姿态：
- 低位模式
- 高位模式

## 3. 梅林地图模型

### 3.1 地图尺寸与高度

梅林区域为 `4 行 x 3 列`。

各格高度为：

- 第 1 排：`400, 200, 400`
- 第 2 排：`200, 400, 600`
- 第 3 排：`400, 600, 400`
- 第 4 排：`200, 400, 200`

### 3.2 当前粗坐标系

当前代码内采用的粗坐标模型为：

- `+X`：朝梅林更深处前进
- `+Y`：朝地图左侧
- 锚点：
- 第 1 排第 2 列中心点 = `(3100, 1470)`
- 格子间距：
- `1200 mm`

实现位置：

- `APP/merlin_map/merlin_map.cpp`

### 3.3 heading 定义

当前离散朝向定义：

- `0 = PosX`
- `1 = PosY`
- `2 = NegX`
- `3 = NegY`

对应理解：

- `0`：车头朝世界坐标 `X+`
- `1`：车头朝世界坐标 `Y+`
- `2`：车头朝世界坐标 `X-`
- `3`：车头朝世界坐标 `Y-`

### 3.4 yaw 目标映射

当前 waypoint 目标角度与 heading 的关系是：

- `0 -> 0 deg`
- `1 -> 90 deg`
- `2 -> 180 deg`
- `3 -> -90 deg`

这样做的原因是：

- 旋转 90 度后再按 `Y / A`
- 目标姿态不会被重新强行拉回 `0 度`

## 4. 传感器布局与作用

### 4.1 laser1

- 前向激光测距
- 主要用于“上台阶是否已经贴近台阶”的判定

当前上台阶触发区间：

- `430 ~ 500 mm`

### 4.2 laser2

- 安装在前腿
- 朝下看
- 主要用于“下台阶高位什么时候回低位”的判定

当前下台阶回低位区间：

- `300 ~ 400 mm`

### 4.3 laser3

- 第二个前向激光测距
- 底层驱动和 `laser1` 不同
- 但对上层暴露出的接口故意做成一致
- 当前和 `laser1` 一起用于上台阶触发

当前上台阶触发区间：

- `450 ~ 500 mm`

### 4.4 后部光电门

- 安装在底盘后部
- 不做距离测量
- 只做动作切换判定

当前作用：

- 上台阶时：
- 高位推进后，决定什么时候收腿回低位

- 下台阶时：
- 低位靠近楼梯边缘后，决定什么时候进入高位

### 4.5 位姿来源

- `x / y`：由上位机 / 雷达持续下发
- 当前底盘实际角度控制：以 IMU / 陀螺仪为准

注意：

- 雷达发来的 yaw 目前可以存，但不作为底层实际锁角主依据
- 真正的当前角度控制仍以 `nav_control::current_yaw` 为核心

## 5. 当前按键行为

当前实测可用的按键逻辑：

- `Xbox`
- 移动到梅林入口前的初始待命点

- `LB`
- 逆时针旋转 90 度

- `RB`
- 顺时针旋转 90 度

- `View`
- 按当前 `x / y` 识别所在梅林格子

- `Y`
- 沿当前车头方向上一个台阶

- `A`
- 沿当前车尾方向下一个台阶

- `LS`
- 在手动辅助里切换“上台阶辅助 / 下台阶辅助”模式

- `RS`
- 打开 / 关闭手动激光辅助

## 6. 当前上下台阶主逻辑

### 6.1 上台阶

当前 `Y` 的总体逻辑为：

1. 识别当前所在格子
2. 根据当前 `heading` 计算贴近楼梯的粗定位点
3. 朝粗定位点移动
4. 如果激光 / 传感器提前触发：
- 直接进入高位
5. 如果到粗定位点还没触发：
- 继续以小速度向前爬，直到触发
6. 记录触发时刻的当前位置，作为动态参考点
7. 进入高位
8. 2006 带整车向前推进，目标为“收腿点”
9. 如果回低位判据提前触发：
- 立即收腿回低位
10. 如果到粗收腿点还没触发：
- 继续高位慢速前进直到触发
11. 回低位
12. 移动到下一个格子中心

### 6.2 下台阶

当前 `A` 的总体逻辑为：

1. 识别当前所在格子
2. 根据当前 `heading` 反方向计算下台阶边缘粗定位点
3. 朝边缘点移动
4. 如果下高位判据提前触发：
- 直接进入高位
5. 如果到粗定位点还没触发：
- 继续低速向后爬，直到触发
6. 进入高位
7. 2006 带整车向后退，目标为“回低位点”
8. 如果 `laser2` 提前满足回低位条件：
- 立刻回低位
9. 如果到粗回低位点还没触发：
- 继续高位慢退直到触发
10. 回低位
11. 若存在下一低一级格子：
- 移动到该格中心
12. 若不存在：
- 保持当前坐标，不再跳回旧测试点

## 7. 当前判据层分工

判据层实现位于：

- `APP/stair_assist/stair_assist.cpp`

当前规则总结：

- 上台阶进入高位：
- `laser1` 命中近距离区间
- 或 `laser3` 命中近距离区间
- 或 `laser2` 命中上高位辅助区间

- 上台阶高位回低位：
- 后部光电门触发

- 下台阶进入高位：
- 后部光电门触发

- 下台阶高位回低位：
- `laser2` 命中 `300 ~ 400 mm`

当前稳定帧数：

- `1 帧`

当前测距触发频率：

- 约 `20 Hz`
- 约每 `50 ms` 触发一次

## 8. 关键常量

- 下台阶从中心退到高位触发粗点：`280 mm`

## 10. 当前已验证结论

当前已经由实车验证通过：

- 4 x 3 梅林全流程可跑通
- 旋转 90 度后仍能继续上下台阶
- `View` 能识别当前梅林格子
- `Y / A` 都已经不是单纯固定直线测试，而是和 `heading` 绑定
- 上下台阶都已经具备：
- 提前触发就提前切动作
- 到粗定位还没触发就继续小速度找触发
- 出梅林边界后不会跳回旧直线测试点

## 11. 当前仍未完成或仍可优化部分

- 梅林外完整任务链路还没做完
- 上位机全自动任务编排还没完整接入
- 出梅林后的专用外部路点体系还没补齐
- 高位推进阶段 2006 的细腻速度规划仍可继续优化
- 若机械高度继续调整，laser / photogate 阈值需要重新整定

## 13. 后续建议

后续新窗口继续完善时，推荐优先顺序：

1. 先补出梅林后的目标点与动作逻辑
2. 再补更高层的任务序列与夹取流程
3. 再优化高位推进速度规划
4. 最后在机械高度稳定后重新整定所有阈值
